DevOps Stack
MediTrack is deployed with a production-grade DevOps pipeline. Every tool below has a specific role in keeping the system reliable, scalable, and observable.
🐳
Docker
Packages the MediTrack API and frontend into isolated containers. Ensures identical behaviour across dev, staging, and hospital production servers. Multi-stage build keeps the image minimal and secure.
docker-compose up --build
☸️
Kubernetes
Orchestrates containers in production. Runs 2 replicas with rolling updates (zero downtime). HorizontalPodAutoscaler scales from 2–10 pods when patient load spikes. Liveness probes auto-restart unhealthy pods.
kubectl apply -f k8s/
🔧
Jenkins
Automates the CI/CD pipeline on every Git push. Runs tests → builds Docker image → pushes to Docker Hub → deploys to Kubernetes. Only deploys to production on the main branch.
:8080 — Jenkins Dashboard
🏗️
Terraform
Provisions AWS EC2 infrastructure as code. Creates the server, security group (ports 22/80/3000), and installs Docker automatically. Remote state in S3 enables team collaboration and audit trails.
terraform apply
📊
Prometheus
Scrapes /metrics from the API every 15 seconds. Tracks HTTP request counts, request durations, active patient counts, and critical patient gauges — all custom metrics exposed by the Node.js app.
:9090 — Prometheus UI
📈
Grafana
Visualises all Prometheus metrics in real-time dashboards. Auto-provisioned with Prometheus as a datasource. Hospital IT staff can see request rates, error rates, uptime, and patient count trends.
:3001 — Grafana Dashboard
